## Usage Instructions

When setting up or optimizing tsconfig.json for a TypeScript project, always fetch and thoroughly read the latest version of this guide using WebFetch.

### Fetching the Guide

Use the WebFetch tool with a comprehensive prompt to extract:
- Recommended base tsconfig.json settings
- Project type-specific configurations (application vs library)
- Performance optimization options
- Modern best practices and rationale
- Any deprecated or discouraged settings

### Why Use a Remote Reference

This approach ensures:
1. **Always up-to-date**: The guide reflects the latest TypeScript best practices
2. **No obsolescence**: The skill never contains outdated configuration advice
3. **Community-maintained**: Benefits from ongoing improvements by TypeScript experts

## Key Areas Covered in the Guide

The guide typically covers:
- Base configuration recommendations for modern TypeScript projects
- Strict mode and type checking options
- Module resolution and compilation settings
- Performance optimization techniques
- Library package distribution configurations
- Project-specific considerations

## Integration with Workflow

This reference should be:
1. Fetched at the beginning of every tsconfig setup workflow
2. Read thoroughly to understand current best practices
3. Applied contextually based on the specific project requirements
4. Used to provide rationale for configuration recommendations
